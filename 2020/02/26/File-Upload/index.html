MIME 文件类型

MIME 一般是根据文件后缀名来判断的，会在发给服务器的请求包中添加，以告知服务器如何去解析处理这个文件，通常由 Content-Type 字段指定，以 Low 难度上传时的抓包截图为例，如图

可以看到，这里的 MIME 类型为 application/octet-stream，如果上传一个 png 图片呢？

再提供一种有趣的绕过思路，不过是 php 老版本的漏洞

%00 截断

php 版本在 5.3.4 以下时，形如 test.php%00.jpg 的字符串因为 chr(0) 的存在会被提前截断，变成 test.php

然而在上传时，因为文件后缀为 .jpg，Content-Type 就会是正常的 image/jpeg

实际传到服务器那边去和文件夹名称一拼接，就变成了 文件夹/test.php，.jpg被截断

因为该环境的 php 版本是 5.4.45，所以无法进行演示，感兴趣的朋友可以自行搜索了解

总的来说，该防御形同虚设，来看看 High 做了什么改善吧

现在又有一个新问题出现了，当我们直接访问传上去的这张 test2.png，服务器并不会将它按 php 文件来处理，只能单纯的显示这张图，给它传参也不会被解释

源码

名字处理方面，impossible 考虑将上传的文件名与一个唯一的随机字符串拼接再做 md5 哈希，这样 %00 截断就失效了

同时生成了一个上传文件的 tmp 副本，方便后面处理

第一个 if 首先检查文件后缀、MIME 类型、文件大小，这些都满足后，会将上传的图片根据文件结构自己再生成一遍（二次渲染）

那些不属于图片结构的东西在 tmp 副本中不会被保留，也就是说我们附加在文件末尾的恶意脚本会被剔除，最后通过 rename 函数将文件移动到指定位置
